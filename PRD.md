# RoamQuest 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 产品定位
**RoamQuest** 是一款 AI 驱动的城市探索应用，通过智能生成个性化的城市体验清单，帮助用户发现城市中的景点、美食和隐藏宝藏。用户可通过拍照打卡记录旅程，并生成精美的旅行报告分享给朋友。

### 1.2 产品 Slogan
> Discover cities, one adventure at a time.

### 1.3 核心价值
| 价值点 | 说明 |
|--------|------|
| 智能化 | AI 自动生成个性化城市体验清单 |
| 便捷性 | 自动位置检测，一键生成清单 |
| 社交化 | 照片打卡，生成精美报告分享 |
| 激励性 | 进度追踪，完成打卡获得成就感 |

### 1.4 目标用户
- **主要用户**：25-45 岁的城市旅行者、探险爱好者
- **使用场景**：旅行规划、城市探索、周末休闲、本地发现
- **平台**：iOS（主要）、Android、macOS、Web（部分功能受限）

## 2. 商业模式

### 2.1 收费策略
采用 **Freemium（免费增值）模式**

| 用户类型 | 权限说明 |
|----------|----------|
| 免费用户 | 每个分类 1 次免费打卡（4 个分类：Landmark、Food、Experience、Hidden）|
| Premium 用户 | 无限次打卡，解锁所有城市 |

### 2.2 订阅方案

| 方案 | 价格 | 节省 | Product ID |
|------|------|------|------------|

| 季付 | $13.99/季 | 7% | `com.roamquest.subscription.quarterly` |
| 年付 | $47.99/年 | 20% | `com.roamquest.subscription.yearly` |

### 2.3 支付方式
- Apple In-App Purchase（iOS）
- Google Play Billing（Android，预留）
- 支持自动续订

## 3. 功能模块详解

### 3.1 用户认证模块

#### 3.1.1 登录方式
- **主要方式**：Sign in with Apple
- **自动登录**：App 启动时检测登录状态，已登录用户直接进入首页

#### 3.1.2 用户资料管理
| 字段 | 说明 |
|------|------|
| 用户名 (username) | 自定义用户名 |
| 全名 (full_name) | Apple Sign-In 获取 |
| 头像 (avatar_url) | 支持上传自定义头像 |
| 偏好设置 (preferences) | JSON 格式存储用户偏号 |

#### 3.1.3 登出流程
1. 点击 Profile 页面的 Sign Out 按钮
2. 显示确认对话框
3. 确认后清除用户 ID 和本地存储
4. 返回登录页面

---

### 3.2 城市探索模块

#### 3.2.1 自动位置检测
- 基于 GPS 获取用户当前位置
- 反向地理编码：经纬度 → 某市名称
- 失败处理：显示错误提示，引导用户手动选择

#### 3.2.2 手动选择城市
- **热门城市列表**：展示预设的热门城市
- **城市搜索**：支持按名称搜索城市
- **底部面板**：CitySelectionBottomSheet 弹出选择

#### 3.2.3 AI 清单生成
- **API**：DeepSeek AI API（替代 Claude）
- **生成内容**：20+ 项清单，包含 4 个分类
- **分类**：
  - Landmark（地标景点）
  - Food（美食推荐）
  - Experience（独特体验）
  - Hidden Gems（隐藏宝藏）

- **输出格式**：纯 JSON
- **模板缓存**：首次生成后保存为模板，后续直接使用

#### 3.2.4 清单历史
- 显示最近 5 个清单
- 按创建时间倒序排列
- 显示每个清单的完成进度

## 4. 用户流程图

### 4.1 首次使用流程

```
启动 App
    ↓
检测登录状态
    ↓
[未登录] → 显示登录页 → Apple Sign In → 首页
[已登录] → 首页
```

### 4.2 生成清单流程（已更新）

```
首页
    ↓
选择城市（自动检测 / 手动选择）
    ↓
[已有未完成的清单] → 直接打开该清单 → 清单页
[无未完成的清单] → 查询模板
    ↓
    [已有模板] → 使用模板 → 清单页
    [无模板] → 调用 AI API → 生成清单 → 保存模板 → 清单页
```

### 4.3 打卡流程

```
清单页
    ↓
点击未完成项目
    ↓
[检查免费额度]
    ↓
[有额度] → Checkin 页面 → 拍照/选图 → 上传 → 保存 → 清单页
[无额度] → 支付墙弹窗 → [订阅] → [取消]
```

### 4.4 生成报告流程

```
清单页
    ↓
点击报告按钮
    ↓
生成视觉报告
    ↓
[分享] / [保存相册]
```

## 6. 技术规格

### 6.1 技术栈

| 分类 | 技术 | 版本 |
|------|------|------|
| 前端框架 | Flutter | >= 3.0.0 |
| 编程语言 | Dart | >= 3.0.0 |
| 状态管理 | Provider | ^6.1.1 |
| 后端服务 | Supabase | ^2.3.4 |
| AI API | DeepSeek | - |
| 认证 | Sign in with Apple | ^6.1.1 |
| 支付 | In-App Purchase | ^3.1.13 |
| 位置服务 | geolocator | ^12.0.0 |
| 地理编码 | geocoding | ^3.0.0 |

### 6.2 支持平台

| 平台 | 支持状态 | 限制说明 |
|------|----------|----------|
| iOS | 完全支持 | 主要平台 |
| Android | 完全支持 | - |
| macOS | 完全支持 | - |
| Web | 部分支持 | 位置服务、相机、Apple Sign In 受限 |

### 6.3 安全策略

- **RLS（行级安全）**：用户只能访问自己的数据
- **API 密钥**：存储在 `.env` 文件中，不提交到 Git
- **用户隔离**：所有用户数据通过 `auth.uid() = user_id` 验证
